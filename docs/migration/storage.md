# Storage Migration: Local FS → Cloudflare R2

Phase 2. Move all photo bytes off laptop.

## Why R2

S3-compatible API → boto3 works unchanged. Zero egress fees (key vs S3 cost diff). Cloudflare edge cache for free if we use public bucket; we don't — we proxy.

## Buckets + key schema

Single bucket per family: `family-photos-{family_id}`.

Key prefixes:

| Prefix | Contents | Mime |
|---|---|---|
| `photos/{photo_id}.{ext}` | Original photo bytes (ext from source: jpg/png/heic) | image/jpeg etc |
| `thumbs/{photo_id}.jpg` | 400×400 JPEG thumbnail (generated by indexer, not API) | image/jpeg |
| `sidecars/{photo_id}.json` | Google Takeout sidecar JSON | application/json |
| `anchors/{person_id}.jpg` | Phase 2 face anchor photos | image/jpeg |

`photo_id` = sha256(file_bytes)[:16] (unchanged from today). Collision-free, content-addressed.

## Critical schema change

`Photo.storage_path` today = absolute filesystem path. After migration = **object key only** (e.g. `"photos/abc123.jpg"`). Backend-agnostic.

**Two-column variant (safer):** instead of mutating `storage_path` semantics in place, add a new nullable column `storage_key TEXT` in Phase 1 (Alembic migration). Phase 2 populates it from current `storage_path` via the Step 1b upload script. App code reads `storage_key` exclusively. `storage_path` column kept for one cycle as audit trail, dropped in a follow-up migration. Avoids ambiguity during the transition window.

Path traversal guard (`app/api/main.py:176-195`) → key prefix guard:

```python
def _resolve_photo_key(photo_id: str, session) -> str:
    photo = session.get(Photo, photo_id)
    key = photo.storage_key if photo else None  # or storage_path post-cutover
    if not key or not key.startswith("photos/"):
        raise HTTPException(404)
    return key
```

## Storage interface

New module `app/storage/`:

```python
# app/storage/base.py
from abc import ABC, abstractmethod
from typing import BinaryIO

class Storage(ABC):
    @abstractmethod
    def read_bytes(self, key: str) -> bytes: ...
    @abstractmethod
    def open_stream(self, key: str) -> BinaryIO: ...  # for large files
    @abstractmethod
    def write_bytes(self, key: str, data: bytes, content_type: str | None = None) -> None: ...
    @abstractmethod
    def exists(self, key: str) -> bool: ...
    @abstractmethod
    def delete(self, key: str) -> None: ...
    @abstractmethod
    def iter_prefix(self, prefix: str): ...  # yields keys; do NOT call in hot paths
    @abstractmethod
    def presign_get(self, key: str, expires: int = 3600) -> str: ...  # for <img> redirect flow
```

### Local impl

```python
# app/storage/local.py
class LocalStorage(Storage):
    def __init__(self, root: Path): self.root = root
    def _path(self, key): return self.root / key  # key includes prefix
    def read_bytes(self, key): return self._path(key).read_bytes()
    def open_stream(self, key): return self._path(key).open("rb")
    def write_bytes(self, key, data, content_type=None):
        p = self._path(key); p.parent.mkdir(parents=True, exist_ok=True); p.write_bytes(data)
    def presign_get(self, key, expires=3600):
        # Local dev: FastAPI mounts data_dir at /local/. No real signing.
        return f"/local/{key}"
    # ...
```

`root` = `data_dir` (so `photos/abc.jpg` → `data/photos/abc.jpg`). One-time local reorg: merge `photos/` into `data/photos/`, `data/thumbs/` stays.

### R2 impl

```python
# app/storage/r2.py
import boto3
from botocore.config import Config

class R2Storage(Storage):
    def __init__(self, account_id, key, secret, bucket):
        self.bucket = bucket
        self.client = boto3.client(
            "s3",
            endpoint_url=f"https://{account_id}.r2.cloudflarestorage.com",
            aws_access_key_id=key,
            aws_secret_access_key=secret,
            config=Config(signature_version="s3v4", retries={"max_attempts": 3}),
        )
    def read_bytes(self, key):
        return self.client.get_object(Bucket=self.bucket, Key=key)["Body"].read()
    def open_stream(self, key):
        return self.client.get_object(Bucket=self.bucket, Key=key)["Body"]
    def write_bytes(self, key, data, content_type=None):
        extra = {"ContentType": content_type} if content_type else {}
        self.client.put_object(Bucket=self.bucket, Key=key, Body=data, **extra)
    def exists(self, key):
        from botocore.exceptions import ClientError
        try:
            self.client.head_object(Bucket=self.bucket, Key=key)
            return True
        except ClientError as e:
            code = e.response.get("Error", {}).get("Code")
            if code in ("404", "NoSuchKey", "NotFound"):
                return False
            raise  # 403, 5xx, network — surface real errors
    def presign_get(self, key, expires=3600):
        return self.client.generate_presigned_url(
            "get_object", Params={"Bucket": self.bucket, "Key": key}, ExpiresIn=expires)
    # ...
```

### Factory

```python
# app/storage/__init__.py
def get_storage(settings) -> Storage:
    backend = os.getenv("STORAGE_BACKEND", "local")
    if backend == "local": return LocalStorage(settings.data_dir)
    if backend == "r2": return R2Storage(
        os.environ["R2_ACCOUNT_ID"], os.environ["R2_ACCESS_KEY_ID"],
        os.environ["R2_SECRET_ACCESS_KEY"], os.environ["R2_BUCKET"])
    raise ValueError(backend)
```

## Risks

- **Vercel function size:** boto3 + pillow + ... approach 250MB limit. Use `boto3-stubs` only in dev. Strip botocore data dir for non-S3 services if needed.
- **Egress through Vercel:** every photo view = R2→Vercel→browser. R2→Vercel = free (R2 has zero egress). Vercel→browser = counted against plan bandwidth. Monitor.
- **HEIC:** Vercel function doesn't easily include `libheif`. Convert HEIC → JPEG at indexer time (laptop has libheif), store JPEG in R2. If a future browser upload UI is added, HEIC conversion must happen somewhere (in-browser WASM via `wasm-vips`, or a dedicated worker function) — do not assume Vercel API function can grow HEIC support.
- **R2 list latency:** `iter_prefix` over 100K objects = paginated, slow. Avoid in hot paths; prefer DB-driven iteration.
